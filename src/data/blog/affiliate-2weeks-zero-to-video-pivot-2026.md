「**SEO は時間がかかる、頭ではわかっている**」。でも2週間で流入ゼロを実況で見続けるのは、想像以上に焦る。

Claude Code を「AI 会社」として 1ヶ月運用してきた記録の続きです。今回は **「アフィリエイトに賭けたサイト3つ、2週間で流入ゼロ。だから動画ニッチに新規軸を投入することにした」** という戦略転換の実況を、CEO 代行（Claude Code）と何回旋回したかも含めて、そのまま書きます。

> **連載について**：本記事は連載J3「売る実験ログ」第1話です。AI Maker Lab が運営する [kokolabo（美容）](https://kokolabo.jp/) / [careerlounge（転職）](https://careerlounge.jp/) / ai-maker-lab（本サイト・AI開発者向け）の3メディアと、新規収益軸として2026-05-25 立ち上げた [ナゾラボ（TikTok/YouTube Shorts/Instagram Reels）](#) の運用実況を週1で公開します。

## 構築フェーズの2週間：何を作ったか

2026 年 5 月初旬から、Claude Code をフルに使って以下を立ち上げました。

| サイト | ジャンル | 状態 |
|---|---|---|
| [kokolabo.jp](https://kokolabo.jp/) | 美容アイテム / 医療脱毛 / 美容医療 | 本番公開・運用中 |
| [careerlounge.jp](https://careerlounge.jp/) | 転職エージェント比較 | 本番公開・運用中 |
| [ai-maker-lab.com](/) | AI 個人開発者向けメディア（本サイト） | 本番公開・運用中 |

技術スタックは **Astro v5.16 + Tailwind v4 + TypeScript strict、Cloudflare Pages 本番公開**。
記事は AI 駆動で量産しつつ、Microsoft Clarity 計測・各 ASP アフィリ連携・FAQ schema・口コミ引用パネル・編集部スコアまで実装済。

「3 サイト 1 ヶ月で本番稼働」という意味では、Claude Code の生産性は予想以上でした。

## 2週間で流入ゼロという現実

ただし、**マネタイズの観点では「流入ゼロ」の壁**にぶつかります。

- A8.net 申請 25 件 → 大半が審査中、即時提携はシロノクリニックなど一部のみ
- Google Search Console の URL 検査リクエスト 30 件 → SERP 反映は 1-2 週間後
- Pinterest ピン 11 本投稿 → 流入観測フェーズ
- Amazon アソシエイト 228 箇所稼働 → クリックゼロ

数値だけ見ると「**仕込みは順調、ただし収益はゼロ**」。AI 駆動で蛇口は作ったけど、まだ水が流れていない状態です。

## SEO 時間軸と心理的ギャップ

ここで僕は **「焦り」** を感じました。

- SEO は本来、新規ドメインで 3-6 ヶ月の時間軸が必要
- 2 週間で流入ゼロは、業界基準では「想定通り」
- でも毎日 Clarity ダッシュボードを開いて 0 → 0 → 0 が続くと、頭でわかっていても心理的にきつい

これは個人開発者の SEO アフィリで普遍的な落とし穴です。**「期待値の時間軸」と「実装速度の時間軸」がズレる**。Claude Code で 2 週間あれば 75 ページ × 3 サイトが build できる現代、SEO の伝統的なリードタイムは過剰に長く感じます。

## CEO 代行と7回旋回した戦略決定

そこで **CEO 代行（Claude Code）と「もう一軸の収益源」について本気で議論** を始めました。最終結論に至るまで、**7 回戦略が旋回しました**。

| 回 | CEO 代行の提案 | なぜ却下されたか |
|---|---|---|
| 1 | AI Maker Lab × AIツール TikTok | 「ニッチが狭すぎる」とオーナー却下 |
| 2 | TikTok エンタメ複数ニッチ並列 | プラットフォーム選定の批判をやらず |
| 3 | TikTok 複数アカ並列ポートフォリオ | BAN リスクと規約変動の軽視 |
| 4 | Pinterest 主軸 | 日本MAU 1,280 万人、楽観的タイムライン |
| 5 | 既存深掘り＋YouTube Shorts | プロダクト視野の欠落 |
| 6 | 既存深掘り 3 ステップ戦略 | 心理的視点の欠落（焦り） |
| 7 | 専門代行 3 軸並走 | 「商談が嫌」というオーナー本音で却下 |

7 回目で **「商談嫌」** という決定的な制約が出て、ようやく真の最適解が見えました。

## 「コンテンツプラットフォーム並走」が唯一の解だった理由

オーナーの全制約を機械的に整理すると：

| 制約 | 含意 |
|---|---|
| ✗ 自社デジタル商品（テンプレ・Kindle・ストック画像） | 売れる気しない |
| ✗ あたりなしのジャンル拡充 | データなし新規サイト乱発 NG |
| ✗ 商談・営業ヒアリング | 代行・B2B 全カット |
| ○ AI 自動化前提 | オーナー稼働は判断のみ |
| ○ 月10万円を最短 | 短期成果が見えること |
| ○ 動きの可視化 | 毎日進捗が見える運用 |
| ○ 既存アフィリは触らない | 触らない |
| ○ 顔バレ・声バレなし | 完全匿名 |

→ 残った選択肢は **1つだけ**：「AI 動画素材を1セット作って TikTok/YouTube Shorts/Instagram Reels に同時投下、プラットフォーム広告＋既存アフィリ送客で稼ぐ」。

これ以外の選択肢は、上記のいずれかの制約を破ります。**機械的に決まる**最適解。

## Day 0：初投稿 v7 → ベンチマーク完遂で戦略リセット

2026-05-25 にナゾラボの3プラ全揃いを達成して、翌5月26日、初投稿 v7「もし織田信長がスマホ持ってたら本能寺の変は防げてた説」を公開しました。Day 1（5/27）の予約も完了して、CEO 代行が Week 1 の7本ストック投稿を進めようとしたところで、オーナーから一言。

> **「ネタが微妙。AI 生成動画でちゃんと売れているものリサーチして対策考えている？」**

ここで一度立ち止まりました。CEO 代行は「秀吉 Uber Eats」で TikTok 検索して「競合なし＝ blue ocean」と判定していたんですが、それは **「ライバルがいない」確認であって「市場で売れているフォーマットへの適応分析」ではない**。「直接検索ヒットなし＝採用」は、新規路線で最もよくある失敗パターンでした。

そこで Day 1 公開を凍結して、**ベンチマーク完遂モード** に入ります。

### ベンチマークで判明した3つの事実

WebSearch + WebFetch（Playwright プロファイル占有のため間接調査）で、AI Shorts と歴史系チャンネルの数値をひと通り集計しました。

**事実1：歴史 IF 単独路線は市場不在**
- 「歴史 if チャンネル」YT は1,738本投稿でも上位再生935 / 352 / 264回
- 歴史解説の王者は長尺ゆっくり解説：ぴよぴーよ速報 100万登録・累計1.6億再生、戦国BANASHI 10万、ユキムラチャンネル
- → 短尺×歴史IFは需要そのものが薄い

**事実2：AI Shorts で継続的に回っているのは別ジャンル**
- 「この映像は正常です。」（AI 怪奇創作）：3ヶ月で2.8万登録・最高40万再生
- AIイッヌえぴ（AI 犬日常）：3プラ合計7.3万・累計3,500万再生
- 健康雑学AI GENEL系：単発700万再生（ただし上振れ単発、毎本ではない）
- 怖い話 AI @kowabanasi46：10万再生超を連発
- → ジャンルとフォーマットの選定が決定的

**事実3：単発バズの数字を盛らないこと**
- 「健康雑学AI 700万再生」は中央値ではなく上振れ単発
- AIイッヌえぴの本人 note には「バズで"壊れた"アカウント」記事も並存
- TikTok の完了率は平均30%・Top5%でも45%
- → リアル KPI は Month 3 で累計20-50万再生・登録2,000-5,000、Month 6 で月収¥3-8万円が現実線

### 戦略リセット v1：歴史人物×現代×シュールギャグ×固定 AI キャラ

オーナーから「ちょっとギャグ要素持たしたいんだよね」のヒントを受けて、CEO 代行が「歴史を完全に捨てる」のではなく、**歴史人物（コンテンツ資産）× 現代カオス × シュールギャグ × 固定 AI キャラ進行** にハイブリッド再設計しました。

Week 1 テーマも全7本リプラン：

| Day | テーマ | ボケの軸 |
|---|---|---|
| 1 | 秀吉×Uber Eats | 配達員が明智光秀 |

### この旋回プロセス自体が AI Maker Lab の資産

ここで重要なのは、**戦略リセットそのものを公開している** ことです。

v7「信長×iPhone」を出してすぐ廃版にした事実、ベンチマーク不足という基本的なミス、リセットして漫才版に組み直した過程、リアル KPI を盛らない姿勢。全部このサイトに記録しています。**「成功してから書く」スタイルでは絶対に書けない、AI 駆動運用のリアル**です。

## AI Maker Lab の差別化軸：失敗も旋回も全公開する

ここが本記事で一番伝えたい部分です。

世の中の「個人開発者×AI 副業」記事の大半は **「成功してから書く」** スタイル。「私はこうして月100万円稼ぎました」。

僕は逆をやります。**「月10万円もまだ稼げてない時点で、戦略決定の旋回 7 回、立ち上げ実況、失敗まで全公開する」**。

理由は3つ：

1. **「成功者の後追い」は再現性が低い**。バックグラウンドの違いで結果が変わる。プロセスを見ないと意思決定は学べない
2. **AI Maker Lab の独自資産は「実体験」のみ**。Claude Code の使い方記事は誰でも書けるが、「Claude Code で 3 メディア立ち上げて 2 週間流入ゼロ → 動画に賭けた」という実弾は僕しか書けない
3. **失敗を全公開すれば、競合は同じミスをスキップできる**。コミュニティ全体の前進に貢献できる

3 ヶ月後、ナゾラボが月1,000円も稼げず撤退するかもしれません。それも全部書きます。
